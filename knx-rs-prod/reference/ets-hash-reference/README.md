# ETS Hash Reference Tool

Generates ApplicationProgram hash prebytes using the **original ETS6 DLLs** via
reflection. Used to create golden test vectors for the Rust `knx-prod` hasher.

> **The ETS DLLs are proprietary and NOT included in this repository.**
> You must supply them yourself from an ETS6 installation.

## Setup

### 1. Obtain the DLLs

Copy these five files from an ETS6 installation directory into `dlls/`:

```
dlls/
├── Knx.Ets.Common.dll
├── Knx.Ets.Common.Types.dll
├── Knx.Ets.Xml.ObjectModel.dll
├── Knx.Ets.Xml.RegistrationRelevanceInformation.dll
└── Knx.Ets.XmlSigning.dll
```

Typical ETS6 install path on Windows:
`C:\Program Files (x86)\ETS6\`

### 2. Adjust the target framework

Edit `ets-hash-reference.csproj` and set `<TargetFramework>` to match your
installed .NET SDK version (run `dotnet --version` to check):

```xml
<TargetFramework>net10.0</TargetFramework>  <!-- or net8.0, net9.0, ... -->
```

### 3. Build

```sh
dotnet build
```

## Usage

```sh
# Compute hash and fingerprint
dotnet run -- path/to/application.xml

# Also dump the pre-MD5 byte stream (for golden vector tests)
dotnet run -- path/to/application.xml output_prebytes.bin
```

### Example

```
$ dotnet run -- ../../tests/fixtures/leakage_app.xml /tmp/leakage_prebytes.bin
MD5:         dd03e07cbe5cc31594a44bea21f566af
Fingerprint: DDAF
Prebytes:    11297 bytes
Wrote 11297 bytes to /tmp/leakage_prebytes.bin
```

### Extracting application XML from .knxprod files

A `.knxprod` file is a ZIP archive. The application XML is inside:

```sh
# List contents
unzip -l product.knxprod

# Extract (the path looks like M-XXXX/M-XXXX_A-YYYY-ZZ-FFFF.xml)
unzip -p product.knxprod "M-0083/M-0083_A-*.xml" > application.xml
```

## How it works

The tool uses .NET reflection to invoke the internal
`ChildElementHashGenerator` class from `Knx.Ets.XmlSigning.dll` as a black box.
This is the same code path ETS uses to compute the ApplicationProgram hash.

The `DynamicValueResolver` is stubbed to write parameter values as plain strings
(`BinaryWriter.Write(string)`). This matches the behavior for standard product
databases generated by OpenKNXproducer and MDT tools.

## Notes

- The tool calls the ETS DLLs via reflection without modification.
- The pre-MD5 byte stream output is used to create golden test vectors
  for verifying the Rust reimplementation byte-for-byte.
